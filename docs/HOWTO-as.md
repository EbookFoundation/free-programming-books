* [Pull request সমূহৰ বিষয়ে](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/about-pull-requests)
* [এটা pull request সৃষ্টি কৰা](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request)

* [YouTube - নবীনসকলৰ বাবে GitHub টিউটৰিয়েল](https://www.youtube.com/watch?v=0fKg7e37bQE)

আনকি যদি আপুনি এজন অভিজ্ঞ মুক্ত উৎস অৱদানকাৰী, তথাপিও কিছুমান কথা আছে যিয়ে আপোনাক বিভ্ৰান্ত কৰিব পাৰে। আপুনি আপোনাৰ PR দাখিল কৰাৰ পিছত, ***GitHub Actions* এ এটা *linter* চলাব, প্ৰায়ে spacing বা alphabetization ৰ সাৰু সমস্যা বিচাৰি পায়**। যদি আপুনি এটা সেউজীয়া বুটাম পায়, সকলো পৰ্যালোচনাৰ বাবে সাজু; কিন্তু যদি নহয়, বিফল হোৱা check ৰ তলত "Details" ত ক্লিক কৰি জানি লওক linter টোৱে কি ভাল পোৱা নাই, আৰু সমস্যাটো সমাধান কৰি আপোনাৰ PR খোলা শাখাত নতুন commit এটা যোগ কৰক।

অৱশেষত, যদি আপুনি নিশ্চিত নহয় যে আপুনি যোগ কৰিব বিচৰা সম্পদটো `Free-Programming-Books` ৰ বাবে উপযুক্ত হয়, [CONTRIBUTING](CONTRIBUTING.md) *([অনুবাদসমূহো](README.md#translations) উপলব্ধ)* ত থকা নিৰ্দেশনাসমূহ পঢ়ক।
